# Улучшения надёжности декодера Telescope

## Проблема

При декодировании реальных кадров (`capture`) обнаружены следующие проблемы:
1. **Бордер не находится** - 28 из 31 кадров не смогли определить калибровочный бордер (0% совпадения)
2. **Неверная сигнатура заголовка** - 3 кадра нашли бордер, но не смогли распарсить заголовок
3. **Отсутствие гарантий целостности** - CRC проверка не работает корректно в многокадровом режиме

## Выполненные улучшения

### 1. Улучшенный поиск бордера (`decoder.go`)

#### Было:
- Поиск с шагом `pixelSize * 2` (минимум 4 пикселя)
- Порог валидности: >70% совпадений
- Остановка при первом найденном варианте

#### Стало:
- **Двухпроходный алгоритм**:
  - Первый проход: строгий порог 80%
  - Второй проход: расслабленный порог 50%
- **Уменьшенный шаг сканирования**: `pixelSize` (минимум 2 пикселя)
- **Полный перебор** всех возможных позиций бордера
- **Выбор лучшего совпадения** вместо первого попавшегося

```go
// findBorderExhaustive ищет бордер полным перебором
func (d *Decoder) findBorderExhaustive(img image.Image, pixelSize format.PixelSize) (point, bool, float64) {
    step := int(pixelSize)
    if step < 2 {
        step = 2
    }
    
    // Сканируем ВСЕ возможные позиции
    for y := 0; y <= height-borderPx*2; y += step {
        for x := 0; x <= width-borderPx*2; x += step {
            _, matchPct := d.validateBorderAtOffset(img, x, y, borderPx, int(pixelSize))
            if matchPct > bestMatch {
                bestMatch = matchPct
                bestOffset = point{x, y}
            }
        }
    }
    
    return bestOffset, bestMatch >= 0.7, bestMatch
}
```

### 2. Учёт JPEG-артефактов

Валидация бордера теперь учитывает небольшие отклонения яркости из-за JPEG-сжатия:

```go
// Было: точное сравнение
if int(gray) == expected

// Стало: допуск на артефакты сжатия
if int(gray) == expected || (gray > 240 && expected == 255) || (gray < 15 && expected == 0)
```

## Рекомендации по дальнейшему улучшению

### 1. Исправление многокадрового CRC
**Проблема**: CRC рассчитывается только для данных текущего кадра, но не учитывает номер кадра.

**Решение**: Включить `frameNum` и `totalFrames` в расчёт CRC:
```go
func (h *Header) SetCRC(data []byte, frameNum uint32) {
    h.CRC = crc32.ChecksumIEEE(append(data, byte(frameNum)))
}
```

### 2. Поддержка Dense Mode
**Проблема**: Декодер не распознаёт сигнатуру заголовка в dense mode.

**Решение**: 
- Проверить порядок чтения нибблов (high/low)
- Добавить логирование промежуточных значений
- Создать отдельный тест для dense mode

### 3. Улучшение детекции режима
**Проблема**: Режим определяется по одному пикселю, что ненадёжно.

**Решение**: 
- Анализировать несколько пикселей мета-зоны
- Использовать статистический подход
- Добавить эвристики для сложных случаев

### 4. Обработка повреждённых кадров
**Проблема**: При ошибке декодирования кадр полностью отбрасывается.

**Решение**:
- Попытка восстановления части данных
- Логирование позиции ошибки
- Опция "best effort" декодирования

## Инструкция по использованию

### Сборка
```bash
cd /workspace
go build -o telescope-decode.exe ./cmd/decode
```

### Запуск декодера
```bash
./telescope-decode.exe -i capture -o decoded --force
```

**Флаги**:
- `-i` - входная директория с кадрами
- `-o` - выходная директория
- `--force` - пропустить CRC проверку (для повреждённых кадров)

### Запуск тестов
```bash
go test ./internal/codec/... -v
```
